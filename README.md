# Table Sync & Excel Export

一个 Chrome 插件：在网页右侧中间提供两个按钮：

- 同步飞书：将当前页面主表格写入飞书多维表格
- 导出Excel：将当前页面主表格下载为 Excel 文件（`.xls`）

## 1. 安装

1. 打开 Chrome `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择目录：`feishu-bitable-sync`

## 2. 飞书配置（首次）

1. 打开插件弹窗，点击“打开飞书设置”
2. 默认只填并保存：
   - `飞书表格完整链接`
   - `应用密钥 App Secret`（来自飞书开放平台应用，不是机器人 Webhook 密钥）
3. 首次若未配置过 App ID，请在“高级设置”补充一次：
   - `App ID`
4. 其余高级项（`user_id_type`、`ignore_consistency_check`）按默认即可

建议勾选：
- 自动创建缺失字段
- 写入来源页面和抓取时间
- 同步前去重（检测到重复记录则跳过）
- 跳过“操作/Action”列

## 3. 使用方式

1. 打开目标网页
2. 如页面支持分页，请先手动选择需要的每页条数，例如 `100条/页`
3. 使用页面右侧中间按钮，或插件弹窗按钮：
   - `同步飞书`
   - `导出Excel`
4. 更新或重新加载扩展后，需要刷新目标网页，否则旧页面脚本可能出现 `Extension context invalidated.`

## 4. 导出说明

- 导出为 `.xls`（HTML 表格格式，Excel 可直接打开）
- 自动包含两列元信息：`来源页面`、`抓取时间`

## 5. 同步说明

- 同步前会尝试点击表头的“查看手机号”眼睛图标，尽量取消手机号隐藏。
- 支持虚拟滚动表格采集：对于只渲染可视行的表格，会滚动表格容器并按 `aria-rowindex` 去重排序。
- 对抖音来客线索表会自动映射常用字段：
  - `姓名` -> `微信备注名称`
  - `线索创建时间` -> `线索时间`
  - `电话/手机号/手机/联系电话` -> `手机号`
  - 电话单元格中的 `所在城市：山西-长治` -> `所在城市`
  - `跟进员工` -> `销售`
  - `最新跟进记录` -> `首次触达情况`
  - `线索阶段` -> `跟进阶段`
  - `来源` 固定写入 `付费流`
- 如果启用“自动创建缺失字段”，同步前会在飞书多维表格里创建缺失字段（文本类型）。
- 如果关闭自动创建字段且目标表缺少业务字段，会直接报错，不再只写入已有字段导致半同步。
- 默认启用同步前去重，通常会使用 `手机号` 等字段检测重复记录。

## 6. 注意

- 插件不会自动翻页，也不会自动设置 `100条/页`；请在网页上手动选择每页条数。
- 如果已经同步过错误数据，重新同步可能会因为去重被跳过。建议先删除错误记录后重试，或临时关闭去重但会产生重复行。
- `App Secret` 存在本地浏览器存储；生产场景建议改服务端中转。
- 新版 `batch_create` 请求已携带 `client_token`、`ignore_consistency_check`、`user_id_type`。
